Positioning & Product
01 — Theme & Atmosphere

Quiet confidence

DivPulse is a dark-theme dividend income dashboard for dividend investors — a deliberate third lane between spreadsheet-cold (Bloomberg-terminal density with no warmth) and consumer-cute (Robinhood-style confetti and mascot energy). Financial dashboards that go too far in either direction lose the user's trust.

Dark canvas
This is a serious tool, not a toy.
One restrained green
Used only where it means something — confidence without shouting.
Generous whitespace
An 8px rhythm — the interface trusts the data enough not to decorate it.
Primary surfaces
Dashboard
Portfolio total, dividend calendar, diversification, recent payments.
Holdings
Full position table — ticker, shares, price, change, yield.
Dividend Alerts
The payment feed. The "receipt" card is the one flourish in the system.

05 — Components

Buttons, icon buttons & inputs

Buttons — dark text on green, not white-on-green
Icon-only buttons — no text labels (standing preference)
On small screens, label repositions above the icon rather than being removed.
Fixed in the 2026-07-14 audit — button and icon-button borders use --text-tertiary (~3.4–4.1:1), not --border-subtle (~1.1–1.4:1), so default-state edges clear the WCAG 1.4.11 3:1 floor.

Motion, Voice & Accessibility
06–09 — Motion · Voice · Accessibility

One flourish, a calm voice, verified contrast

Motion — restraint is the point
The receipt card is the only component with a designed entrance (slide-up + fade, 500ms) plus a 600ms count-up. Every animation has a prefers-reduced-motion fallback.
Voice — calm, precise, quietly confident
DoDon't
"Coca-Cola Co. · +$46.20""🎉 You just earned $46.20!"
"Couldn't refresh prices.""Oops! Something went wrong 😅"

Implementation & Closing
11 — Implementation Notes

Reusing this system

Import order — colors_and_type.css → kit/tokens.css → kit/components.css → kit/interactions.js.
Two-tier tokens — prefer --role-* semantic aliases in new code over raw palette tokens.
Dark-only by design — no light mode exists; treat any request as new brand work, not inversion.

This guide is generated from real evidence — brand-spec.md, design-system.html, and kit/ — and kept in sync with the audited, current package.
