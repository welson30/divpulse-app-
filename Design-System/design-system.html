PaidPrime Design System

Quiet confidence for dividend investors — the third lane between spreadsheet-cold and consumer-cute.

Prepared for dividend investors Status reference document Updated Jul 13, 2026 Version 1.0

Financial dashboards fail in two directions: spreadsheet-cold or consumer-cute. PaidPrime holds a third lane.

Dark backgrounds signal a serious tool. Restrained green signals growth without shouting. Generous whitespace signals that we trust the data enough not to decorate it.

01 — Color

Color system

Layered depth instead of flat black, one growth green used only where it means something, and red reserved strictly for loss data. Click a swatch to copy its hex.

Backgrounds — layered depth
Base#09090B — canvas
Surface#1C1C1E — cards, panels
Elevated#252528 — modals, dropdowns
Primary green — growth, brand, positive
Red — loss data only, never decorative
Neutrals — text hierarchy
Semantic accents — sparing use
Rule of thumb: green appears only where it means something — gains, active states, primary buttons, success. Red never decorates; it reports a loss.
Accessibility note — text-tertiary (#7D7D85) clears 4.5:1 against bg-base alone (4.9:1) but drops to 4.2/3.7/3.4:1 on bg-surface/bg-elevated/bg-hover — under the AA floor for normal text on every surface a card or modal actually sits on. Reserve it for the type specimen and disabled states; use text-secondary for anything a user needs to reliably read, including timestamps and captions inside cards. Its exact value is tuned for double duty as the interactive-control border (3:1 WCAG 1.4.11 floor) — see brand-spec.md §Neutrals before changing it.

04 — Elevation & depth

No shadows — lightness does the work

Drop shadows vanish on dark surfaces, so depth is layered lightness instead: base → surface → elevated, each defined by a 1px hairline border.

Surface#1C1C1E
